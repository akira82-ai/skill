# Auto Skill 需求规范

## 功能需求

### FR1: 需求分析
- 系统必须能从用户输入中提取关键词
- 系统必须能分类用户意图（文档/分析/管理/处理/创作）

### FR2: 本地匹配
- 系统必须扫描 `~/.claude/skills/` 和 `.claude/skills/`
- 系统必须读取每个 skill 的 SKILL.md 元数据
- 系统必须计算匹配度分数

### FR3: 远程搜索
- 系统必须支持多源搜索（Firecrawl、Web Reader、Web Search Prime）
- 系统必须从 skills.sh 获取 skill 信息
- 系统必须提供安装命令

### FR4: 用户交互
- 所有需要确认的操作必须使用 AskUserQuestion
- 呈现的选项必须清晰、简洁
- 必须提供取消选项

### FR5: 安装与执行
- 系统必须能自动安装 skill
- 系统必须能使用 Skill tool 执行
- 系统必须处理安装失败的情况

## 非功能需求

### NFR1: 性能
- 本地匹配应在 2 秒内完成
- 远程搜索应在 10 秒内完成

### NFR2: 可靠性
- 最多 3 次重试机会
- 失败时提供明确的错误信息

### NFR3: 用户体验
- 所有操作必须有清晰的反馈
- 选项描述应简洁明了
