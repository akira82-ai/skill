## 搜索策略

### 1. 智能判断是否需要搜索

**需要搜索的场景：**
- 技术性、专业性强的主题
- 需要数据支撑的观点
- 时事热点和最新趋势
- 需要权威性增强的内容
- 不熟悉的领域

**不需要搜索的场景：**
- 纯个人经历和感悟
- 创意写作和想象内容
- 用户明确表示不需要搜索
- 极其简单和常识性内容

### 2. 关键词提取

从用户输入中提取：
```
核心概念 + 领域限定词 + 时间限定词

例如：
"WebAssembly的发展前景"
→ 核心概念：WebAssembly
→ 领域限定：Web development
→ 时间限定：2024/2025 trends
```

### 3. 搜索查询构建

**多角度搜索：**

| 角度 | 查询示例 |
|------|----------|
| 基础信息 | "什么是[主题]" |
| 最新趋势 | "[主题] 2024/2025 趋势" |
| 数据研究 | "[主题] research/study" |
| 案例实践 | "[主题] case study/examples" |
| 问题解决 | "how to [动词] [主题]" |
| 对比分析 | "[主题A] vs [主题B]" |

### 4. 来源筛选标准

**高质量来源特征：**
- ✓ 权威机构发布
- ✓ 有明确作者
- ✓ 提供数据/引用来源
- ✓ 内容深度专业
- ✓ 发布时间较新

**低质量来源特征：**
- ✗ 内容农场
- ✗ 无作者信息
- ✗ 纯广告内容
- ✗ 信息过时
- ✗ 标题党

### 5. 信息综合方式

**数据引用：**
```
根据 [来源] 的研究，[具体数据]。

例如：根据 McKinsey 2024年的报告，AI可以将企业效率提升40%。
```

**案例补充：**
```
[公司/人物] 的案例说明了 [观点]。

例如：Netflix 的推荐系统案例说明了机器学习在用户体验中的应用。
```

**对比分析：**
```
[观点A]，但 [观点B] 提供了不同的视角。

例如：一些人认为AI会取代工作，但世界银行的研究显示AI更多是增强而非取代。
```

**趋势引用：**
```
行业趋势显示，[趋势描述]。

例如：Gartner 预测，到2025年，50%的企业将采用云原生架构。
```

## 信息整合到推文

### 整合策略

1. **开头引用**：用震撼数据/研究开头吸引注意
2. **观点支撑**：用研究数据支撑你的观点
3. **案例丰富**：用真实案例增强说服力
4. **趋势展望**：结合行业趋势提升前瞻性

### 参考资料格式

在推文末尾附上参考资料：

```markdown
---
**参考资料：**

1. [论文/报告标题], [作者/机构], [年份]
2. [文章标题], [媒体名称], [日期]
3. [文档名称], [官方来源]
```

### 引用注意事项

- 不要过度引用，适量即可
- 引用要准确，不要曲解原意
- 标注清楚来源
- 优先引用最新来源（1-2年内）
- 多元化来源，不要只引用一个

## 质量检查清单

搜索到的信息在使用前检查：

- [ ] 来源是否权威可靠
- [ ] 内容是否最新（近1-2年）
- [ ] 数据是否有明确来源
