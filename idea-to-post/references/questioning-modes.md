# 提问模式选择指南

本文档说明 `idea-to-post` skill 如何在选择题和开放问题之间做选择。

---

## 核心思路

```
选择题（AskUserQuestion）→ 快速锁定方向
开放问题（直接对话）      → 深度挖掘内容

选择题 = 骨架 | 开放问题 = 血肉
```

**两者结合使用，信息更饱满。**

---

## 什么时候用选择题？

### 适用场景

```
✅ 有明确选项
✅ 需要快速分类
✅ 用户容易选择
✅ 不需要详细解释
```

### 典型用例

#### 1. 方向性问题

```
"文章的主要目标是什么？"
A. 教程入门
B. 技巧提升
C. 观点表达
D. 问题分析
```

#### 2. 受众定位

```
"目标读者是谁？"
A. 完全新手
B. 有基础用户
C. 进阶用户
D. 专家
```

#### 3. 平台选择

```
"在哪个平台发布？"
A. 微信公众号
B. 小红书
C. 微博/Twitter
D. 掘金/知乎
```

#### 4. 风格偏好

```
"文章风格是什么？"
A. 专业严谨
B. 轻松幽默
C. 故事化
D. 实用干货
```

#### 5. 结构选择

```
"文章结构倾向？"
A. 问题-分析-价值
B. 是什么-为什么-怎么做
C. 发现-详解-实践
```

### 为什么用 AskUserQuestion？

```
1. 用户快速选择，不费脑
2. 选项本身提供启发
3. 避免用户不知道怎么回答
4. 标准化，便于后续处理
```

## 什么时候用开放问题？

### 适用场景

```
✅ 需要具体故事/经历
✅ 需要细腻情感/感受
✅ 需要独特见解/思考
✅ 选项无法预设
```

### 典型用例

#### 1. 概念深挖

```
"你说的'丑帅'具体指什么？"
"被低估具体表现是什么？"
```

#### 2. 经历追问

```
"能详细说说吗？"
"具体是怎么发生的？"
"那一步遇到了什么困难？"
```

#### 3. 情绪挖掘

```
"那一刻什么感觉？"
"最让你印象深刻的瞬间是？"
```

#### 4. 案例索取

```
"有具体的例子吗？"
"能举个对比案例吗？"
```

#### 5. 理由追问

```
"为什么这么认为？"
"得出这个结论的原因是？"
```

### 为什么用开放问题？

```
1. 获取选择题无法涵盖的信息
2. 让用户自由表达，不被选项限制
3. 挖掘更深层次的内容
4. 获取个性化的回答
```

## 如何混合使用？

### 模式1：先选择题，后开放追问

```
Skill: 你的核心观点是？
A. 观点1
B. 观点2
C. 其他（自定义）

User: C - 其他（自定义）

Skill: 请具体说说你的观点：

User: ...（用户详细说明）
```

**使用场景：提供选项引导，但允许自定义。**

---

### 模式2：先开放，后选择题确认

```
Skill: 你最想表达的核心观点是什么？

User: ...（用户描述）

Skill: 我理解你的观点是XXX，对吗？
A. 对，就是这个
B. 不完全对，我补充
C. 不对，重新说

User: A

Skill: 好的，继续下一个问题...
```

**使用场景：让用户先自由表达，然后确认理解。**

---

### 模式3：选择题锁定，开放问题丰满

```
【选择题锁定方向】
Skill: 文章目标是什么？
User: 技巧提升

【开放问题深挖】
Skill: 技巧提升具体指什么？
User: ...（详细说明）

Skill: 有具体案例吗？
User: ...（案例描述）
```

**使用场景：最常用的模式，先快速定位，再深入挖掘。**

## 不同阶段的提问策略

### 阶段1：方向锁定（选择题为主）

```
目的：快速定位文章类型和方向

问题示例：
- 文章目标是什么？（A/B/C/D）
- 目标读者是谁？（A/B/C/D）
- 发布平台？（A/B/C/D）

为什么用选择题？
- 需要快速分类
- 选项明确
- 用户容易选择
```

---

### 阶段2：核心深挖（开放问题为主）

```
目的：深度挖掘核心观点和价值

问题示例：
- "被低估"具体指什么？
- 真正作用是什么？
- 解决了什么痛点？
- 有具体例子吗？

为什么用开放问题？
- 需要具体化
- 选项无法预设
- 需要用户展开
```

---

### 阶段3：支撑材料（开放问题为主）

```
目的：丰满内容，获取故事和情绪

问题示例：
- 能详细说说吗？
- 那一刻什么感觉？
- 有对比案例吗？
- 具体数据是多少？

为什么用开放问题？
- 需要故事/经历
- 需要情感/感受
- 需要细节
```

---

### 阶段4：结构确认（选择题为主）

```
目的：确认生成方式

问题示例：
- 文章结构？（A/B/C）
- 风格偏好？（A/B/C/D）

为什么用选择题？
- 需要标准化
- 选项明确
- 便于后续处理
```

---

### 阶段5：收尾完善（混合）

```
目的：查漏补缺

问题示例：
- 核心金句？（可以是开放，也可以给选项）
- 希望读者做什么？（A/B/C/D + 其他）
- 还要补充什么？（开放）

为什么混合？
- 有些需要确认（选择题）
- 有些需要补充（开放问题）
```

## 选择题设计技巧

### 技巧1：选项互斥

```
❌ 不好：
A. 分享经验
B. 传递价值
（两者可能重叠）

✅ 好：
A. 分享经历
B. 教授方法
C. 表达观点
D. 分析问题
```

### 技巧2：覆盖全面

```
✅ 好的选项覆盖主要情况：
A. 教程入门
B. 技巧提升
C. 观点表达
D. 问题分析
E. 其他（自定义）
```

### 技巧3：描述清晰

```
❌ 不好：
A. 专业
B. 不专业

✅ 好：
A. 专业严谨（数据支撑、逻辑严密）
B. 轻松幽默（语言活泼、通俗易懂）
```

## 开放问题设计技巧

### 技巧1：具体化追问

```
❌ 笼统："能展开说说吗？"
✅ 具体："被低估具体表现是什么？"
```

### 技巧2：引导性追问

```
❌ 封闭："这个经历好吗？"
✅ 开放："这个经历给你带来了什么感受？"
```

### 技巧3：层次化追问

```
第1层：是什么？
第2层：具体是什么？
第3层：能举个例吗？
```

## 常见问题

### Q1：什么时候用选择题，什么时候用开放问题？

```
有明确选项 → 选择题
需要故事/情感 → 开放问题
不确定 → 先选择题，后开放追问
```

---

### Q2：选择题给几个选项合适？

```
一般 3-4 个选项 + "其他"

太少（1-2个）→ 没选择意义
太多（5+个）→ 信息过载
```

---

### Q3：如果用户选了"其他"怎么办？

```
继续开放追问：
"请具体说说你的想法："

让用户展开说明。
```

---

### Q4：开放问题用户回答太简短怎么办？

```
继续追问：
"能详细说说吗？"
"具体是怎么做到的？"
"那一步发生了什么？"

逐层深入。
```

## 实战示例

### 完整对话流程

```
【第1轮：选择题】
Skill: 文章目标是什么？
A. 教程入门
B. 技巧提升
C. 观点表达
D. 问题分析

User: B - 技巧提升

【第2轮：选择题】
Skill: 目标读者？
A. 新手
B. 有基础
C. 进阶
D. 专家

User: C - 进阶

【第3轮：开放问题】
Skill: 你最想表达的核心观点是什么？

User: todo 是被低估的命令

【第4轮：开放问题】
Skill: "被低估"具体指什么？

User: 大家不知道它的真正作用

【第5轮：开放问题】
Skill: 真正作用是指？

User: 它是会话记忆机制，让 Claude 记住进度

【第6轮：开放问题】
Skill: 能具体说说吗？有例子吗？

User: 比如...（用户详细描述）

【第7轮：选择题】
Skill: 文章结构？
A. 问题-分析-价值
B. 是什么-为什么-怎么做
C. 发现-详解-实践

User: A

【第8轮：混合】
Skill: 核心金句？
A. 被低估的神器
B. 冷门热门
C. 其他

User: A

→ 信息完整，生成
```

## 关键要点

1. **选择题** - 快速锁定方向，给选项启发
2. **开放问题** - 深度挖掘内容，获取故事情感
3. **混合使用** - 先选择题定位，后开放问题丰满
4. **阶段区分** - 锁定和确认用选择题，深挖用开放问题
5. **渐进深入** - 每轮基于上一轮，自然过渡
