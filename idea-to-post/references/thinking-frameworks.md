# 思考框架详细说明

本文档详细说明 `idea-to-post` skill 中集成的各种思考框架。

---

## 1. 5W1H分析法

### 概述

5W1H是最全面的信息收集框架，确保覆盖问题的各个维度。

### 六个维度

| 维度 | 问题 | 目的 |
|------|------|------|
| What | 是什么？ | 明确核心概念/问题/产品 |
| Why | 为什么？ | 探究原因、动机、意义 |
| Who | 谁？ | 确定主体、受众、相关方 |
| When | 何时？ | 时间维度、时机选择 |
| Where | 何地？ | 空间维度、场景定位 |
| How | 如何做？ | 方法、路径、实施 |

### 适用场景

- 需要全面介绍一个主题
- 项目启动和规划
- 复杂问题的拆解分析
- 不确定用哪个框架时的默认选择

### 推文示例结构

```
【What】AI是什么？一个简单的定义
【Why】为什么AI重要？它带来的变革
【Who】谁在用AI？从个人到企业
【When】现在是AI时代吗？时机分析
【Where】AI应用在哪些领域？
【How】如何开始使用AI？
```

## 2. SCQA讲故事模型

### 概述

SCQA是麦肯锡咨询常用的叙事结构，非常适合讲故事型内容。

### 四个阶段

| 阶段 | 含义 | 内容 |
|------|------|------|
| S (Situation) | 情境 | 描述当前稳定状态/背景 |
| C (Complication) | 冲突 | 出现的问题/变化/挑战 |
| Q (Question) | 疑问 | 如何解决这个问题？ |
| A (Answer) | 答案 | 提供解决方案/洞察 |

### 适用场景

- 讲述产品/项目从0到1的故事
- 分享问题解决经历
- 职场成长故事
- 创业历程分享

### 推文示例结构

```
【S-情境】一切看起来都很正常，我们按部就班地工作...

【C-冲突】直到有一天，用户投诉暴涨30%，系统崩溃了...

【Q-疑问】为什么会出现这个问题？我们该如何应对？

【A-答案】经过深入分析，我们发现了根本原因...并找到了解决方案...
```

### 使用技巧

- 情境要让读者产生共鸣
- 冲突要有戏剧性，制造紧张感
- 疔问要自然引出答案
- 答案要有意外感或启发性

## 3. 黄金圈法则 (Golden Circle)

### 概述

Simon Sinek 提出的经典模型，从内向外思考：Why → How → What

### 三个层次

| 层次 | 问题 | 说明 |
|------|------|------|
| Why (内核) | 为什么？ | 信念、初衷、使命、存在的意义 |
| How (中层) | 如何做？ | 方法论、流程、实现方式 |
| What (外层) | 做什么？ | 具体的产品、服务、结果 |

### 适用场景

- 品牌故事和理念表达
- 产品价值主张
- 个人/团队使命介绍
- 创业初心分享

### 推文示例结构

```
【Why-为什么】我们相信每个人都应该拥有掌控自己数据的能力...

【How-如何做】因此我们构建了一套端到端加密的去中心化存储系统...

【What-做什么】最终呈现给用户的，是一个简单安全的私人云盘...
```

### 使用技巧

- 从Why开始，这是最有力量的部分
- How要展示独特的方法论
- What是结果，但不是重点
- 大多数人从What说起，你从Why说起会更突出

## 4. AIDA营销模型

### 概述

经典的营销漏斗模型，引导读者从注意到行动。

### 四个阶段

| 阶段 | 目标 | 方法 |
|------|------|------|
| Attention | 吸引注意 | 震撼数据、反常识观点、悬念 |
| Interest | 激发兴趣 | 相关性、痛点共鸣 |
| Desire | 产生欲望 | 展示利益、社会认同 |
| Action | 促使行动 | 明确行动指引、紧迫感 |

### 适用场景

- 产品推广
- 课程/活动宣传
- 转化型文案
- 营销活动

### 推文示例结构

```
【Attention-注意】90%的人都在用错误的方式学习编程...

【Interest-兴趣】你花了大量时间却进展缓慢，这不是你不够聪明...

【Desire-欲望】想象一下，如果有一套方法能让你3个月内独立开发项目...

【Action-行动】点击链接，免费领取我的编程学习路线图...
```

### 使用技巧

- Attention要足够震撼
- Interest要触动痛点
- Desire要描绘美好愿景
- Action要具体明确

## 5. PREP观点表达法

### 概述

简洁有力的观点表达结构，适合发表看法和评论。

### 四个部分

| 部分 | 内容 | 说明 |
|------|------|------|
| Point | 观点 | 直接表明你的核心观点 |
| Reason | 理由 | 解释为什么持有这个观点 |
| Example | 例子 | 用具体案例支撑 |
| Point | 总结 | 重申观点，强化印象 |

### 适用场景

- 表达专业观点
- 评论热点事件
- 分享经验心得
- 职场感悟

### 推文示例结构

```
【Point-观点】远程办公是未来的趋势，企业应该积极拥抱。

【Reason-理由】远程办公可以降低成本、提高效率、扩大人才选择范围...

【Example-例子】我们公司全面远程办公后，人均产出提升了40%，离职率下降了50%...

【Point-总结】所以，不要抗拒远程办公，而要思考如何把它做得更好。
```

### 使用技巧

- Point要开门见山，不绕弯子
- Reason要逻辑清晰
- Example要具体可感
- 结尾Point可以呼应开头，略有升华

## 6. 5-Why深度分析法

### 概述

丰田生产方式的核心方法，通过连续追问找到根本原因。

### 方法

对每个问题连续问5次"为什么"，层层深入直到找到根本原因。

```
问题：机器停了
为什么？ - 因为保险丝断了
为什么？ - 因为负荷过大
为什么？ - 因为轴承枯竭
为什么？ - 因为油泵故障
为什么？ - 因为油泵没有过滤器（根本原因）
```

### 适用场景

- 问题根因分析
- 失败复盘
- 深度技术分析
- 产品问题剖析

### 推文示例结构

```
【问题】我们的用户留存率一直上不去

【Why 1】为什么？用户用一次就不再回来了
【Why 2】为什么？因为他们找不到核心功能
【Why 3】为什么？因为产品导航太复杂
【Why 4】为什么？因为我们在做加法而不是减法
【Why 5】为什么？因为我们缺乏"少即是多"的设计哲学（根本原因）

【结论】问题不在用户，而在我们的产品理念
```

### 使用技巧

- 每一层都要诚实回答
- 不要停留在表面原因
- 找到可行动的根本原因
- 可以少于5次，如果已找到根本原因

## 7. 第一性原理

### 概述

埃隆·马斯克推崇的思考方式，从最基本的事实出发，拆解到本质再重构。

### 三个步骤

| 步骤 | 说明 |
|------|------|
| 1. 拆解 | 把问题拆解到最基本的事实/真理 |
| 2. 质疑 | 质疑所有假设和"常识" |
| 3. 重构 | 从基本原理出发，重新构建解决方案 |

### 适用场景

- 创新突破
- 颠覆性思考
- 技术架构设计
- 产品创新

### 推文示例结构

```
【拆解-基本事实】电池的本质是什么？是金属钴、镍、铝、碳的聚合物...

【质疑-打破假设】市场说电池很贵，这是真的吗？如果直接从原材料购买...

【重构-新方案】实际上原材料成本只需要80美元/kWh，远低于市场价600美元...

【结论】所以我们可以自己造电池，成本降低80%
```

### 使用技巧

- 区分"事实"和"观点"
- 不被"大家都这么做"束缚
- 问"这在物理上可能吗？"
- 从物理学/经济学第一性原理出发

## 8. FBA产品介绍法

### 概述

产品营销的经典结构，从特性到利益再到优势。

### 三个层次

| 层次 | 内容 | 示例 |
|------|------|------|
| Feature | 特性 | 产品有什么功能 |
| Benefit | 利益 | 这个功能给用户带来什么 |
| Advantage | 优势 | 与其他产品相比有何不同 |

### 适用场景

- 产品介绍
- 功能发布
- 服务说明
- 对比评测

### 推文示例结构

```
【Feature-特性】我们的笔记软件支持双向链接

【Benefit-利益】这意味着你可以建立知识网络，让想法产生连接，发现隐藏的关联...

【Advantage-优势】不同于传统笔记软件的文件夹结构，这是像大脑一样的工作方式...
```

### 使用技巧

- Feature要简洁具体
- Benefit要从用户视角出发（"你获得"）
- Advantage要突出差异化
- 可以有多个Feature-Benefit-Advantage组合

## 框架选择决策树

```
你的灵感
    │
    ├─ 包含"为什么/本质/使命" → 黄金圈
    ├─ 包含"问题/挑战/故事" → SCQA
    ├─ 包含"推广/转化/营销" → AIDA
    ├─ 包含"观点/看法/建议" → PREP
    ├─ 包含"深挖/根因/底层" → 5-Why
    ├─ 包含"创新/突破/颠覆" → 第一性原理
    ├─ 包含"产品/功能/优势" → FBA
    └─ 其他/不确定 → 5W1H
```

---

## 框架组合使用

对于复杂内容，可以组合多个框架：

- **5W1H + SCQA**：先全面了解背景，再讲故事
- **黄金圈 + FBA**：讲清楚理念，再介绍产品
- **AIDA + PREP**：先吸引注意，再表达观点
- **5-Why + 第一性原理**：找到根本原因，再重构方案

不要被框架限制，框架是工具，不是目标。
