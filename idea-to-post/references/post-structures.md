# 推文结构指南

本文档详细说明不同平台的推文结构和写作技巧。

---

## 通用推文结构

所有平台推文都应包含的基本要素：

```
┌─────────────────────────────────────┐
│           标题 (Title)              │  ← 吸引人的标题
├─────────────────────────────────────┤
│           钩子 (Hook)               │  ← 抓住注意力的开头
├─────────────────────────────────────┤
│                                     │
│          主体 (Body)                │  ← 核心内容，按框架展开
│                                     │
├─────────────────────────────────────┤
│         行动召唤/结尾 (CTA)          │  ← 促使行动或总结
├─────────────────────────────────────┤
│           标签 (Tags)               │  ← 相关话题标签
└─────────────────────────────────────┘
```

## 微信公众号

### 平台特点

- **字数限制**：2000字以上，深度长文
- **阅读场景**：沉浸式阅读，专注度高
- **读者期望**：有深度、有干货、结构清晰
- **传播方式**：朋友圈转发、社群分享

### 结构模板

```markdown
# 【标题】主标题 | 副标题

**摘要/导语**：一句话概括核心观点，激发阅读兴趣
```

### 写作技巧

1. **标题技巧**
   - 数字法：《7个xxx技巧》
   - 疑问法：《为什么xxx？》
   - 对比法：《从xxx到xxx》
   - 金句法：《xxx：一句话总结》

2. **排版技巧**
   - 使用多级标题（# ## ###）
   - 重要内容加粗
   - 使用引用块（>）突出金句
   - 适当使用分隔线（---）

3. **配图建议**
   - 开头一张吸引人的封面图
   - 每个章节配相关插图
   - 数据信息用图表展示
   - 结尾配行动召唤图

4. **结构技巧**
   - 一个核心观点贯穿全文
   - 每个部分一个小标题
   - 每段不超过5行
   - 用"总-分-总"结构

## 小红书

### 平台特点

- **字数限制**：500-1000字，实用干货
- **阅读场景**：快速浏览，碎片化阅读
- **读者期望**：实用、可操作、有干货
- **传播方式**：算法推荐、搜索发现

### 结构模板

```markdown
【标题】🔥震惊/必看/干货！xxx的N个方法

一张吸引眼球的封面图

---

📌 前言：一句话痛点 + 解决方案

你是不是也遇到过xxx的情况？今天分享我的xxx方法，亲测有效！

---

✨ 方法一：xxx

👉 具体操作：
1️⃣ 第一步
2️⃣ 第二步
3️⃣ 第三步

💡 小贴士：注意xxxx

---

✨ 方法二：xxx

👉 具体操作：
...

---

📊 效果展示：

使用前：xxx
使用后：xxx

---

💡 总结/核心要点

1. 要点一
2. 要点二
3. 要点三

---

🔗 相关推荐

#话题1 #话题2 #话题3

💬 评论区见！有问题欢迎留言~

---

【收藏起来慢慢看】
【点赞让更多人看到】
```

### 写作技巧

1. **标题技巧**
   - 加emoji吸引眼球 🔥
   - 用"震惊"、"必看"等词
   - 强调干货属性
   - 数字吸引：《5个xxx技巧》

2. **emoji使用**
   - ✨ 方法/技巧
   - 👉 操作步骤
   - 💡 小贴士/要点
   - 📊 数据/对比
   - 🔥 热门/重点
   - 💬 互动引导

3. **排版技巧**
   - 每个方法用分隔线分开
   - 关键词加粗
   - 用数字列表展示步骤
   - 留白充足，不要密集

4. **内容要点**
   - 直接给方法，少废话
   - 可操作性要强
   - 配前后对比图
   - 强调效果

5. **标签策略**
   - 3-5个相关话题标签
   - 包含热门标签
   - 包含精准标签
   - 标签放在文末

## 微博/Twitter

### 平台特点

- **字数限制**：140-280字，极简表达
- **阅读场景**：快速刷过，注意力极短
- **读者期望**：快速获取信息，一句话精华
- **传播方式**：转发、点赞、话题聚集

### 结构模板

```
【金句型】
一句话核心观点 + 简短解释 + 话题标签

【数据型】
震撼数据 + 一句话解读 + 话题标签

【观点型】
观点 + 理由 + 结论 + 话题标签

【提问型】
问题 + 思考 + 开放结尾 + 话题标签

【行动型】
痛点 + 解决方案 + 行动召唤 + 话题标签
```

### 具体示例

```markdown
# 金句型

90%的人都不知道的效率秘诀：不是做得更多，而是做得更少。
专注最关键的20%，产出80%的价值。
#效率 #生产力

---

# 数据型

震惊！研究发现：每天工作4小时的人，产出比工作8小时的人高出30%。
质量胜过数量。
#工作效率 #研究

---

# 观点型

我认为远程办公是未来趋势。
理由：降低成本、提高效率、扩大人才池。
建议：企业应该积极拥抱，而非抗拒。
#远程办公 #工作方式

---

# 提问型

为什么越努力越焦虑？
或许我们努力的方向错了。
你有什么看法？
#思考 #人生

---

# 行动型

还在用低效的方式学习？
试试费曼学习法：教你别人 = 自己学会。
今天就开始吧！
#学习方法 #费曼技巧
```

### 写作技巧

1. **开头抓眼球**
   - 用数字开头
   - 用提问开头
   - 用反常识观点开头
   - 用痛点开头

2. **一句话核心**
   - 每条推文一个核心观点
   - 一句话能说完不说两句
   - 删掉所有冗余词

3. **金句化表达**
   - 对比：从xxx到xxx
   - 排比：xxx，xxx，xxx
   - 比喻：xxx就像xxx
   - 警句：xxx不是xxx，而是xxx

4. **话题标签**
   - 1-3个相关标签
   - 标签在末尾
   - 包含热门话题
   - 包含垂直话题

5. **配图策略**
   - 一张图胜过千言万语
   - 信息图展示数据
   - 金句卡片
   - 前/后对比图

### 平台特点

- **字数限制**：1000-1500字，专业职场
- **阅读场景**：职业发展时间，相对专注
- **读者期望**：专业见解、行业洞察、职场经验

```markdown
## 核心观点/洞察

### 第一点

**观点阐述**

解释观点，提供专业见解

【案例】相关的职业案例或行业案例

### 第二点

3. 建议3：具体说明

## 个人思考/反思

分享个人的职业思考

> "回顾我的职业道路，我认为..."

## 结尾 + 行动召唤

总结要点，邀请讨论

"你对这个问题怎么看？欢迎在评论区分享你的看法"

---

#行业话题 #职业发展 #专业领域
```

### 写作技巧

1. **标题技巧**
   - 专业术语：如《xxx的最佳实践》
   - 行业热点：结合当前行业趋势
   - 经验总结：《xxx年，我学到的xxx》
   - 方法分享：《如何xxx》

2. **专业表达**
   - 使用行业术语
   - 引用数据和报告
   - 提及知名公司/案例
   - 展示专业思考

3. **结构清晰**
   - 用小标题分隔内容
   - 用数字列表展示要点
   - 用引用块突出观点
   - 逻辑严密

4. **职场相关**
   - 结合职业经历
   - 分享行业洞察
   - 提供职场建议
   - 引发职业思考

5. **互动引导**
   - 结尾提问引发讨论
   - 鼓励分享经历
   - 征求意见建议
   - 建立专业联系

## 跨平台适配策略

### 同一内容，多平台发布

```
微信公众号（长文）
      ↓ 精简
   小红书（干货版）
      ↓ 提炼
   微博（金句版）
      ↓ 调整
 LinkedIn（专业版）
```

### 适配原则

| 原内容 | 公众号 | 小红书 | 微博 | LinkedIn |
|--------|--------|--------|------|----------|
| 深度分析 | ✓完整 | ✓精简 | ✗删除 | ✓保留 |
| 案例故事 | ✓完整 | ✓简短 | ✗删除 | ✓保留 |
| 数据引用 | ✓详细 | ✓简略 | ✓一句话 | ✓引用 |
| 操作步骤 | ✓详细 | ✓完整 | ✗删除 | ✓保留 |
| 个人观点 | ✓完整 | ✓保留 | ✓核心 | ✓专业 |
| 金句 | ✓保留 | ✓保留 | ✓主推 | ✓保留 |

### 内容分层

```
第一层：核心观点（所有平台）
    ↓
第二层：支撑理由（公众号、LinkedIn、小红书）
    ↓
第三层：详细案例（公众号、LinkedIn）
    ↓
第四层：操作方法（公众号、小红书）
```

## 通用写作技巧

### 标题公式

1. **数字法**：《N个xxx技巧》
2. **疑问法**：《如何xxx？》《为什么xxx？》
3. **对比法**：《从xxx到xxx》《xxx vs xxx》
4. **秘诀法**：《xxx的秘密》《不为人知的xxx》
5. **警告法**：《警惕xxx》《xxx的误区》
6. **承诺法**：《30天学会xxx》《让你xxx》

### 开头技巧

1. **故事开头**："我遇到了一个问题..."
2. **数据开头**："90%的人都..."
3. **提问开头**："你是否曾经..."
4. **金句开头**："xxx不是xxx，而是xxx..."
5. **痛点开头**："你是不是也..."

### 结尾技巧

1. **总结回顾**："综上所述..."
2. **行动召唤**："现在就..."
3. **开放提问**："你怎么看..."
4. **展望未来**："未来..."
5. **金句升华**："记住，xxx..."

### 排版原则

1. **留白充足**：不要密集堆砌
2. **层次分明**：用标题分隔
3. **重点突出**：加粗/引用
4. **视觉引导**：用符号/emoji
5. **段落短小**：每段3-5行
