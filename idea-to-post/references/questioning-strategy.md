# 持续渐进式提问策略

本文档说明 `idea-to-post` skill 如何设计持续追问的问题。

**核心思路：框架内化 + 选择题锁定方向 + 开放问题丰满内容**

---

## 核心原则

### 原则1：框架内化，不外显

**运用框架思维设计问题，但不说"我在用某某框架"。**

```
❌ 错误示范：
"我推荐用 PREP 框架，现在问 Point：你的观点是什么？"

✅ 正确做法：
"你最想表达的核心观点是什么？"
（问题带有 Point 思维，但对话自然）
```

**为什么？**
- 用户不关心框架，只想把想法表达清楚
- 机械套用框架会让对话生硬
- 框架是思维工具，不是提问模板

---

### 原则2：选择题 + 开放问题结合

```
选择题（AskUserQuestion）→ 快速锁定方向
开放问题（直接对话）      → 深度挖掘内容

选择题 = 骨架 | 开放问题 = 血肉
```

| 问题类型 | 适用场景 | 示例 |
|----------|----------|------|
| **选择题** | 有明确选项、快速分类 | "目标是什么？A.教程 B.技巧 C.分析" |
| **开放问题** | 故事/经历/情感/细节 | "能详细说说吗？" |
| **混合** | 选项 + 自定义追问 | 选择"Other"后继续追问 |

---

### 原则3：渐进深入，动态调整

**每轮问题基于上一轮的回答，自然过渡。**

```
用户："todo 是被低估的命令"
    ↓
追问："被低估"具体指什么？（What 思维）
    ↓
用户："大家不知道它的真正作用"
    ↓
追问：真正作用是什么？（What 思维追问）
    ↓
用户："它是会话记忆机制"
    ↓
追问：解决了什么痛点？（Why 思维）
    ↓
追问：有具体例子吗？（Example 思维）
```

**框架思维融入每个问题，但过渡自然，不是机械按模板走。**

---

### 原则4：基于主题，不问无关

**问题必须紧扣用户输入的核心主题。**

```
✅ 好的问题：
- "丑帅"具体指什么？（紧扣主题）
- skill 架构哪里"丑"？（具体化）
- 这个 skill 帮你做了什么？（验证价值）

❌ 差的问题：
- 你喜欢喝咖啡吗？（与主题无关）
- 今天天气怎么样？（完全无关）
```

---

### 原则5：每轮聚焦 1-2 个问题

**避免信息过载，让用户有思考空间。**

```
❌ 错误：一次问5个
"目标？受众？平台？风格？要强调什么？"

✅ 正确：分多轮
第1轮：目标是什么？
第2轮：受众是谁？
第3轮：平台是什么？
```

## 框架思维融入问题

### PREP 思维融入

```
Point 思维   → "你的核心观点是什么？"
Reason 思维  → "为什么这么认为？理由是？"
Example 思维 → "有没有具体的例子或经历？"
Point 思维   → "你想用一句话总结吗？"
```

### 5W1H 思维融入

```
What 思维 → "这具体指什么？"
Why 思维  → "为什么重要？解决了什么？"
Who 思维  → "目标受众是谁？"
Where 思维→ "在什么场景下？"
When 思维 → "什么时候发生的？"
How 思维  → "怎么做到的？"
```

### SCQA 思维融入

```
Situation 思维    → "最初的状态是怎样的？"
Complication 思维 → "出现了什么冲突或挑战？"
Question 思维     → "产生了什么疑问？"
Answer 思维       → "答案/解决方案是什么？"
```

**框架是思维工具，不是提问模板。用框架思维设计问题，但不说框架。**

## 选择题 vs 开放问题

### 什么时候用选择题（AskUserQuestion）？

```
✅ 适合场景：
- 有明确选项（平台、风格、目标、结构）
- 需要快速分类
- 用户容易选择
- 不需要详细解释

示例：
"目标平台是什么？A.公众号 B.小红书 C.微博"
"文章风格？A.专业严谨 B.轻松幽默 C.故事化"
```

### 什么时候用开放问题（直接对话）？

```
✅ 适合场景：
- 需要具体故事/经历
- 需要细腻情感/感受
- 需要独特见解/思考
- 选项无法预设

示例：
"能详细说说吗？"
"那一刻什么感觉？"
"你为什么这么认为？"
"有具体例子吗？"
```

### 混合使用

```
AskUserQuestion 提供选项 + "Other" 选项
    ↓
如果用户选择 "Other"
    ↓
继续开放追问
```

## 渐进式提问流程

### 阶段1：方向锁定（选择题为主）

```
第1轮：目标是什么？
第2轮：目标受众是谁？
第3轮：发布平台？
```

**目的：快速定位文章类型和方向**

---

### 阶段2：核心深挖（开放问题为主）

```
用户："todo 是被低估的命令"
    ↓
"被低估"具体指什么？（What 思维）
    ↓
真正作用是什么？（What 追问）
    ↓
解决了什么痛点？（Why 思维）
    ↓
有具体例子吗？（Example 思维）
```

**目的：深度挖掘核心观点和价值**

---

### 阶段3：支撑材料（开放问题为主）

```
"能详细说说吗？"
"有对比案例吗？"
"那一刻什么感觉？"
"有没有具体数据？"
```

**目的：丰满内容，获取故事和情绪**

---

### 阶段4：结构确认（选择题为主）

```
"文章结构？A.问题-分析 B.是什么-为什么-怎么做"
"风格偏好？A.专业 B.幽默 C.故事化"
```

**目的：确认生成方式**

---

### 阶段5：收尾完善（混合）

```
"核心金句？"
"希望读者做什么？"
"还要补充什么？"
```

**目的：查漏补缺**

---

## 典型对话示例

### 示例：观点类主题

```
【第1轮：方向锁定】
Skill: 文章目标是什么？
User: 技巧提升
Skill: 目标读者？
User: 进阶用户

【第2轮：核心深挖】
Skill: 你最想表达的核心观点是什么？
User: todo 是被低估的命令
Skill: "被低估"具体指什么？
User: 大家不知道它的真正作用
Skill: 真正作用是指？
User: 它是会话记忆机制，让 Claude 记住进度
Skill: 会话记忆机制解决了什么痛点？
User: 多任务时 Claude 会忘记做到哪了

【第3轮：支撑材料】
Skill: 能具体说说吗？有例子吗？
User: 比如...（用户讲具体经历）
Skill: 那个经历中，最让你感受到 todo 价值的时刻是？
User: 当 Claude 精确说"继续任务3"的时候

【第4轮：结构确认】
Skill: 文章结构倾向？
User: 问题-分析-价值

【第5轮：收尾】
Skill: 核心金句？
User: 被低估的神器
Skill: 希望读者做什么？
User: 关注获取更多

→ 信息完整，生成
```

## 追问技巧

### 1. What 追问 - 具体化

```
用户："效率提升了很多"
追问："具体提升了多少？"
```

### 2. Why 追问 - 挖掘原因

```
用户："这个方案很好用"
追问："为什么这么认为？"
```

### 3. How 追问 - 挖掘方法

```
用户："我们成功了"
追问："怎么做到的？"
```

### 4. Example 追问 - 索取案例

```
用户："这确实是个问题"
追问："有具体例子吗？"
```

### 5. Emotion 追问 - 挖掘情感

```
用户："最后解决了问题"
追问："那一刻什么感觉？"
```

### 6. Contrast 追问 - 对比

```
用户："现在好多了"
追问："和以前比，最大不同是什么？"
```

## 具体化追问技巧（防止用户跳过）

### 问题：追问太抽象，用户容易跳过

```
❌ 太抽象："能具体说说吗？"
✅ 更具体："最近一次遇到'多任务时忘记进度'是什么时候？
           做什么功能？Claude 具体说了什么？"
```

### 技巧1：追问具体时间

```
❌ "有具体例子吗？"
✅ "最近一次是什么时候？上周？上个月？"
```

### 技巧2：追问具体场景

```
❌ "怎么发生的？"
✅ "是在做什么功能的时候？登录？支付？还是其他？"
```

### 技巧3：追问具体反应

```
❌ "当时什么感觉？"
✅ "Claude 说'我做到哪了'那一刻，你是什么反应？
           是无语？是烦躁？还是无奈？"
```

### 技巧4：提供模板降低门槛

```
❌ "能详细说说吗？"
✅ "可以用这个模板说说：
   那天我在做[功能]，
   Claude 说了[具体的话]，
   我当时的感觉是[情绪]"
```

### 技巧5：二选一降低难度

```
❌ "具体是什么场景？"
✓ "是在开发新功能的时候，还是调试 bug 的时候？"
```

### 技巧6：追问最小细节

```
❌ "整个经过是怎样的？"
✓ "Claude 说的第一句话是什么？"
```

## 挖掘情绪价值的追问技巧

### 为什么需要情绪价值？

**自媒体的核心是"人"，不是"信息"。**

读者想看的是：
- 你踩过的坑
- 你崩溃的时刻
- 你发现价值时的惊喜
- 不是冷冰冰的技术分析

### 情绪追问技巧

#### 1. 追问"崩溃时刻"

```
"最让你崩溃/无语/烦躁的时刻是什么？
 具体发生了什么？"
```

#### 2. 追问"惊喜时刻"

```
"发现 todo 真正价值的那一刻，你是什么感觉？
 是'原来如此'的恍然大悟，
 还是'终于找到了'的如释重负？"
```

#### 3. 追问"转折瞬间"

```
"从'觉得没用'到'真香'，中间发生了什么？
 有没有一个具体的瞬间让你改变了看法？"
```

#### 4. 追问"身体感受"

```
"那一刻你有什么身体反应？
 是拍大腿？是长叹一口气？
 还是忍不住笑了？"
```

#### 5. 追问"后续影响"

```
"从那之后，你的使用习惯有什么变化？
 是每次都会用 todo？还是会推荐给别人？"
```

## 挖掘独特性的追问技巧

### 为什么需要独特性？

**避免内容太"标准"，像官方文档。**

读者想看的是：
- 别人没发现的技巧
- 别人没踩过的坑
- 别人没有的使用方式

### 独特性追问技巧

#### 1. 追问"独特发现"

```
"你有没有发现一些官方文档没提到的技巧？
 或者是一些别人都不知道的用法？"
```

#### 2. 追问"独特踩坑"

```
"你有没有踩过别人都没踩过的坑？
 或者遇到一些很奇怪的 bug？"
```

#### 3. 追问"独特用法"

```
"你有没有一些'不按常规'的使用方式？
 比如用 todo 做一些意想不到的事情？"
```

#### 4. 追问"独特观点"

```
"你对 todo 有没有一些'反常识'的理解？
 和大多数人的看法不一样的那种？"
```

#### 5. 追问"独特组合"

```
"你有没有把 todo 和其他命令/工具组合使用？
 产生了一些意想不到的效果？"
```

## 搜索失败的处理策略

### 问题：搜索失败会失去外部佐证

```
如果 WebSearch 失败：
- 内容缺少数据支撑
- 观点缺乏权威性
- 说服力下降
```

### 处理方案

#### 方案1：明确告知并寻求替代

```
"搜索服务暂时不可用。
为了内容的说服力，你有没有：
- 看到过相关的讨论或文章？
- 自己做过一些测试或验证？
- 有一些数据可以分享？"
```

#### 方案2：让用户成为佐证

```
"搜索失败了，但没关系。
你的使用经历本身就是很好的佐证。
你能说说：
- 你用了多久 todo？
- 大概用了多少次？
- 有没有对比过用和不用的情况？"
```

#### 方案3：寻求社区验证

```
"搜索失败了。
你有没有在以下地方看到过相关讨论：
- GitHub Issues
- Reddit
- Twitter/X
- 技术博客

有的话可以分享一下，我整合到内容中。"
```

#### 方案4：降低外部佐证优先级

```
"搜索失败了，没有外部数据支撑。
这会影响内容的权威性，但我们可以：
1. 用你的真实经历来弥补
2. 强调这是个人经验分享
3. 在文末说明"欢迎交流补充"

这样可以吗？"
```

## 信息完整度检查

### 必须完整（缺失则继续追问，不可跳过）

- [ ] **核心观点** - 要表达的观点是什么？
- [ ] **目标受众** - 写给谁看？
- [ ] **发布平台** - 在哪里发？
- [ ] **支撑案例** - 有具体的例子/经历吗？（**自媒体必须**）
- [ ] **情绪价值** - 有什么共鸣点/情绪钩子？（**自媒体必须**）

### 应该完整（尽量追问）

- [ ] **外部佐证** - 有搜索资料可以引用吗？
- [ ] **独特观点** - 有别人没提到的见解吗？

### 最好完整（有则更佳）

- [ ] **互动设计** - 希望读者做什么？
- [ ] **行动召唤** - 有具体的行动指引吗？
- [ ] **风格偏好** - 什么风格？

## 终止追问的时机

### 可以停止

1. **核心维度已完整**（观点、受众、平台、案例、情绪）
2. 用户表现出不耐烦（连续简短回答）
3. 信息出现冗余（新增信息不再带来新维度）
4. 用户明确表示满意

### 必须继续

1. **核心观点模糊** - 不知道你想表达什么
2. **支撑案例缺失** - 没有真实案例/经历（**自媒体必须**）
3. **情绪价值缺失** - 没有共鸣点/情绪钩子（**自媒体必须**）
4. **外部佐证缺失** - 搜索失败且没有替代信息

### 关于案例和情绪的特别说明

**案例和情绪是自媒体内容的灵魂，不能跳过。**

如果用户不愿意提供：
- 明确告知影响："没有真实案例，内容会缺乏真实感"
- 提供引导："可以是一个小片段，不需要完整故事"
- 降低门槛："哪怕是一个瞬间的感受也可以"
- 最后确认："确定不需要案例吗？内容会比较干"

## 迭代优化流程（90 分+ 必备）

### 为什么需要迭代优化？

**初稿往往只有 70-80 分，需要打磨才能达到 90 分+。**

```
初稿生成后：
- 信息可能没有完全整合
- 案例可能不够生动
- 情绪可能不够饱满
- 独特性可能没有突出

需要：反思 → 优化 → 再生成
```

---

### 迭代优化的时机

**在初稿生成后，主动询问：**

```
"内容已生成。在定稿前，我想确认几个问题：

1. 案例部分是否生动？需要补充细节吗？
2. 情绪共鸣是否到位？需要加强吗？
3. 独特观点是否突出？需要强化吗？
4. 还有哪里需要优化？"
```

---

### 迭代优化的方向

#### 1. 案例不够生动

```
"案例部分感觉还可以更生动。
能补充一些细节吗？
- 具体的对话？
- 当时的场景？
- 你的反应？"
```

#### 2. 情绪不够饱满

```
"情绪部分可以再饱满一些。
最让你[崩溃/惊喜]的时刻，
能再详细说说吗？
当时的想法是什么？
身体有什么反应？"
```

#### 3. 独特性不够突出

```
"独特性部分还可以再突出。
这个技巧/观点，
是你独家的发现吗？
还是有别人也提到过？
如果是独家的，能再强调一下吗？"
```

#### 4. 结构可以优化

```
"文章结构可以再优化。
要不要调整一下：
- 把案例放在最前面？
- 把金句做成标题？
- 把对比做成表格？"
```

---

### 迭代优化的示例

```
Skill: 初稿已生成。在定稿前，我想确认：
      案例部分是否生动？需要补充细节吗？

User: 案例有点干，不够具体

Skill: 能补充一些细节吗？
      比如：
      - 是在做什么功能的时候？
      - Claude 具体说了什么？
      - 你当时什么反应？

User: 是做用户登录的时候...
      Claude 说"我做到哪了"
      我当时很无语...

Skill: 好的，我来更新内容，把这些细节加进去。
      （重新生成）

Skill: 更新完成。还要优化其他部分吗？

User: 标题可以更吸引人一点

Skill: 当前标题是"被低估的神器：Claude Code 的 todo 命令"
      想要什么风格的标题？
      A. 悬念式
      B. 数据式
      C. 对比式
      D. 金句式

User: D - 金句式

Skill: 好的，我来重新设计标题...
      （更新标题）

Skill: 最终版本完成。
      要不要再看看还有什么要优化的？

User: 不用了，这样可以

Skill: 好的，最终版本已准备就绪。
```

---

### 迭代优化的原则

1. **主动发起** - 生成后主动询问，不要等用户提
2. **具体建议** - 不问"好不好"，问"哪里需要优化"
3. **提供选项** - 给出具体的优化方向供选择
4. **快速迭代** - 小步快跑，一次优化一个点
5. **确认再结束** - 用户确认不需要优化了再结束

---

### 迭代优化的完整流程

```
初稿生成
    ↓
主动反思：哪里还可以更好？
    ↓
用户反馈：案例有点干
    ↓
具体追问：补充什么细节？
    ↓
重新生成：整合新信息
    ↓
再次反思：还要优化吗？
    ↓
用户反馈：标题不够吸引人
    ↓
提供选项：悬念/数据/对比/金句？
    ↓
重新生成：更新标题
    ↓
最终确认：还有要优化的吗？
    ↓
用户确认：可以了
    ↓
最终版本
```

---

## 关键要点总结

1. **框架内化** - 用框架思维设计问题，但不说框架
2. **混合提问** - 选择题定方向，开放问题丰满内容
3. **渐进深入** - 每轮基于上一轮回答，自然过渡
4. **紧扣主题** - 不问无关问题
5. **每轮聚焦** - 1-2个问题，不过载
6. **完整才停** - 核心维度都有才生成
7. **案例情绪必须** - 自媒体内容必须有真实案例和情绪共鸣
8. **追问要具体** - 防止用户跳过，追问具体时间/场景/反应
9. **搜索失败要应对** - 寻求替代方案，不放弃外部佐证
10. **追求独特性** - 挖掘别人没有的技巧、坑、观点
11. **90分+目标** - 需要7-10轮对话，追求优质自媒体内容
12. **迭代优化** - 初稿后主动反思优化，不满足于"能生成"

## 90 分+ 内容的达成路径

### 对话轮次与内容质量

| 轮次 | 获取内容 | 质量提升 |
|------|---------|---------|
| 1-2 轮 | 方向、平台、受众 | 技术文档水平（60分） |
| 3-4 轮 | 核心观点、价值分析 | 结构清晰（70分） |
| 5-6 轮 | 真实案例、情绪共鸣 | 有血有肉（80分） |
| 7-8 轮 | 独特观点、外部佐证 | 有说服力（85分） |
| 9-10 轮 | 深度打磨、迭代优化 | 优质内容（90分+） |

### 不可跳过的三大支柱

1. **真实案例** - "最近一次什么时候？做什么？说了什么？"
2. **情绪共鸣** - "那一刻什么感觉？身体反应？"
3. **独特观点** - "别人没发现的技巧？反常识的理解？"

### 迭代优化是最后 10 分的关键

初稿 80 分 → 反思优化 → 90 分+
