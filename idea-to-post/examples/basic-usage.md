# 基础使用示例

本文档展示 `idea-to-post` skill 的持续渐进式提问流程。

**核心特点：不是一轮问完，而是持续追问直到信息完整。**

---

## 示例1：观点类主题（PREP框架）

### 用户输入

```
扩展灵感：远程办公是未来
```

### 持续提问对话

**【第1轮：核心观点】**

```
Skill: 你的核心观点是"远程办公必然成为主流"，
        还是"远程办公只适合某些行业"？

User: 只适合某些行业，不是所有行业
```

**【第2轮：理由追问】**

```
Skill: 为什么这么认为？是什么让你得出这个结论？

User: 我所在的公司试过全面远程，结果协作效率下降了
```

**【第3轮：案例深挖】**

```
Skill: 能具体说说吗？哪个环节的效率下降？
        有数据对比吗？

User: 会议沟通效率下降约30%，会议数量反而增加了
```

**【第4轮：反方思考】**

```
Skill: 有没有远程办公效果好的案例或环节？

User: 独立的开发任务效率反而提升了
```

**【第5轮：观点提炼】**

```
Skill: 所以你想表达的是：
        "远程办公适合独立任务，不适合高频协作"？

User: 对，就是这个
```

**【第6轮：目标受众】**

```
Skill: 这篇文章写给谁看？
        A. 正在远程办公的人
        B. 考虑远程办公的公司决策者
        C. 反对远程办公的人
        D. 其他

User: B - 公司决策者
```

**【第7轮：情绪共鸣】**

```
Skill: 你希望读者看完后产生什么感受？
        是"我也要重新思考远程策略"的反思，
        还是"我的判断是对的"的共鸣？

User: 共鸣，让他们知道不是所有行业都适合远程
```

**【第8轮：行动召唤】**

```
Skill: 你希望读者看完后做什么？
        A. 分享自己的远程办公经历
        B. 重新评估公司的远程策略
        C. 转发给需要的人
        D. 其他

User: B - 重新评估公司的远程策略
```

**【第9轮：发布平台】**

```
Skill: 主要在哪个平台发布？
        A. 微信公众号
        B. 掘金/知乎
        C. 微博/Twitter
        D. LinkedIn

User: B - 掘金/知乎
```

**【Skill 判断：信息完整】**

✅ 核心观点：远程办公只适合某些行业
✅ 支撑案例：公司经历 + 具体数据
✅ 目标受众：公司决策者
✅ 情绪共鸣：不是所有人都适合远程
✅ 行动召唤：重新评估策略
✅ 发布平台：掘金/知乎

→ 开始生成内容

---

### 生成结果

#### 掘金/知乎版

```markdown
# 远程办公不是银弹：我们的全面远程实验失败了

## 前言

两年前，我们公司决定全面远程办公。

那时候远程办公正火，很多大厂都在宣传远程的好处。我们也跟风了，以为这是未来。

**一年后，我们取消了全面远程，改成了混合办公。**

---

## 失败的经历

### 问题：协作效率下降

全面远程后，我们发现：

- 会议数量增加了 50%
- 会议沟通效率下降了约 30%
- 很多简单的沟通变得复杂

以前走到工位问一下就能解决的事，现在要约会议、等时间。

### 原因分析

为什么会出现这个问题？

**因为我们这个行业（我们做的是需要频繁协作的产品设计）
不太适合全面远程。**

---

## 成功的部分

但并不是说远程办公完全没用。

**我们发现：独立的开发任务效率反而提升了。**

开发同事反馈：
- 没有办公室打扰，专注时间更长
- 可以灵活安排工作时间
- 省去了通勤时间

---

## 结论

远程办公不是"对或错"的问题，而是"适合与否"的问题。

**适合全面远程的行业/工作：**
- 独立性强的开发任务
- 文案、设计等个人创作
- 客服等标准化工序

**不适合全面远程的行业/工作：**
- 需要频繁创意碰撞的团队
- 需要实时协作的项目
- 面向客户的服务行业

---

## 给决策者的建议

如果你在考虑是否全面远程，请先问自己三个问题：

1. 你的工作性质是否适合远程？
2. 你的团队是否有足够的自律？
3. 你的协作工具是否足够完善？

**不要跟风，要基于自己的实际情况做决策。**

---

*你的公司全面远程效果如何？欢迎分享你的经历。*

#远程办公 #团队管理 #工作方式
```

```
扩展灵感：我们产品从0到1的故事
```

### 持续提问对话

**【第1轮：情境确认】**

```
Skill: 故事的"情境"是什么？最初的状态是怎样的？

User: 我们是一个小团队，想做一款开发者工具
```

**【第2轮：冲突追问】**

```
Skill: 出现了什么"冲突"或挑战？

User: 大厂都在做类似产品，我们找不到差异化
```

**【第3轮：疑问深挖】**

```
Skill: 你们产生了什么关键疑问？如何破局？

User: 我们问自己：大厂不做什么？
```

**【第4轮：答案追问】**

```
Skill: 答案是什么？你们发现了什么机会？

User: 大厂不做小众场景，我们专注于小众但高频的场景
```

**【第5轮：结果确认】**

```
Skill: 最终的结果是什么？

User: 现在在这个细分领域，我们是第一名
```

**【第6轮：反转点】**

```
Skill: 这个过程中，有没有什么特别艰难的时刻？
        想放弃的时候？
```

**【第7轮：转机追问】**

```
Skill: 那后来是怎么转机的？

User: 我们发现用户不用不是因为产品不好，是因为不知道怎么用。
        加了引导后，留存率提升了3倍
```

**【第8轮：核心感悟】**

```
Skill: 这个故事最想让读者感受到什么？
        A. 小团队的逆袭
        B. 差异化的价值
        C. 产品引导的重要性
        D. 其他

User: A + B - 小团队通过差异化也能成功
```

**【第9轮：目标平台】**

```
Skill: 在哪个平台发布？

User: 公众号
```

**【Skill 判断：信息完整】**

→ 开始生成内容

```markdown
# 小团队的破局之路：我们在大厂的阴影下找到了光

## S-情境：起点

三年前，我们是一个5人的小团队。

想做一款开发者工具，但发现大厂都在做类似产品。

**我们感到绝望：怎么和大厂竞争？**

---

## C-冲突：困境

产品做出来了，但没什么人用。

上线3个月，日活还是只有几百。

团队士气低落，有人开始怀疑：
"我们是不是该放弃？"

---

## Q-疑问：破局

那是一个深夜，我们几个创始人坐在会议室。

我们问自己：**大厂不做什么？**

大厂：
- 追求大而全
- 服务大众用户
- 做通用的功能

那我们呢？

---

## A-答案：聚焦

我们发现了机会：**小众但高频的场景。**

大厂不做，因为市场不够大。
但对我们来说，这个"小市场"已经足够养活团队。

我们砍掉了80%的功能，只专注于一个场景：
**开发者的本地调试工具。**

---

## 转折：又一个困境

聚焦之后，用户还是很少。

我们崩溃了：方向对了吗？

后来我们发现问题不在方向，在引导：
**用户不知道怎么用。**

我们加了产品引导、视频教程、示例代码。

**结果：留存率提升了3倍。**

---

## 现在

三年过去了。

在这个细分领域，我们是第一名。

虽然用户量不如大厂的产品，但：
- 用户更精准
- 付费意愿更强
- 我们活下来了

---

## 感悟

小团队不要和大厂正面竞争。

**找到大厂不屑做的，把它做到极致。**

市场小没关系，够你活得很好就行。

---

*你的小团队是如何找到破局点的？欢迎分享。*
```

## 关键要点总结

### 持续提问的特点

1. **不是一轮问完** - 多轮对话，逐步深入
2. **基于主题追问** - 每个问题都紧扣用户输入的主题
3. **基于框架设计** - 运用思考框架的逻辑组织问题
4. **动态调整** - 根据用户回答调整下一轮问题
5. **信息完整才停止** - 核心维度都有才生成

### 提问的层次

```
第1轮：大方向（目标、受众、平台）
    ↓
第2轮：核心要素（基于框架）
    ↓
第3轮：细节补充（案例、数据、反转）
    ↓
第4轮：完善设计（情绪点、行动召唤）
    ↓
第5轮：确认与修正
```

### 信息完整的标准

| 维度 | 检查 |
|------|------|
| 核心观点 | ✅ 清晰 |
| 目标受众 | ✅ 明确 |
| 支撑材料 | ✅ 充分 |
| 情绪价值 | ✅ 有共鸣点 |
| 行动召唤 | ✅ 具体明确 |
| 平台适配 | ✅ 符合特点 |
| 外部佐证 | ✅ 有搜索资料 |
